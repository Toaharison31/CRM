4. CONNEXION NON THREAD-SAFE
------------------------------------------------------------------------

PROBLÈME:
  Fichier: src/databases/Connexion.java
  Ligne: 16
  Sévérité: 🔴 CRITIQUE
  
  ❌ AVANT:
  --------
  public class Connexion {
      private static Connection connection;  // NON THREAD-SAFE!
      
      public static Connection getConnection() throws SQLException {
          if (connection == null || connection.isClosed()) {
              connection = DriverManager.getConnection(...);
          }
          return connection;
      }
  }
  
  Risques:
  - Problème de race condition en multithreading
  - Double-checking locking anti-pattern
  - Multiple threads créent simultanément des connexions
  - Fuite de ressources possibles
  - Comportement imprévisible en environnement multithreadé

SOLUTION APPLIQUÉE:
  ✓ Ajout d'un lock synchronisé (CONNECTION_LOCK)
  ✓ Synchronisation de getConnection() et closeConnection()
  ✓ Gestion robuste des exceptions
  ✓ Logging des erreurs approprié
  
  ✓ APRÈS:
  --------
  public class Connexion {
      private static Connection connection;
      private static final Object CONNECTION_LOCK = new Object();  // LOCK!
      
      public static Connection getConnection() throws SQLException {
          synchronized (CONNECTION_LOCK) {  // THREAD-SAFE!
              if (connection == null || connection.isClosed()) {
                  try {
                      connection = DriverManager.getConnection(
                          Config.getDatabaseUrl(),
                          Config.getDatabaseUser(),
                          Config.getDatabasePassword()
                      );
                      System.out.println("✓ Connexion établie.");
                  } catch (SQLException e) {
                      System.err.println("❌ ERREUR: " + e.getMessage());
                      throw e;
                  }
              }
              return connection;
          }
      }
      
      public static void closeConnection() {
          synchronized (CONNECTION_LOCK) {  // THREAD-SAFE!
              if (connection != null) {
                  try {
                      connection.close();
                  } catch (SQLException e) {
                      System.err.println("❌ ERREUR: " + e.getMessage());
                  } finally {
                      connection = null;
                  }
              }
          }
      }
  }
  
  Avantages:
  ✓ Thread-safe avec synchronisation
  ✓ Une seule connexion créée à la fois
  ✓ Pas de race conditions
  ✓ Gestion robuste des exceptions
  ✓ Logging approprié des erreurs
  ✓ Méthode isConnected() ajoutée pour vérification
